export const environment = [curly_open]
  production: false,
  lembretesApiUrl: 'https://devreminder.herokuapp.com'
[curly_close];

Visão geral

Neste arquivo definimos configurações da aplicação que devem estar acessíveis no ambiente de desenvolvimento. Por exemplo, aqui utilizamos esse arquivo para declarar o endereço de conexão com a API no atributo lembretesApiUrl. Deve-se, ao publicar a aplicação em produção (buildar a aplicação), replicar essas configurações no arquivo environments.prod.ts, alterando os seus valores de acordo com a necessidade do ambiente de produção. Estes arquivos são criados automaticamente pelo Angular, assim como a constante environment, a partir da qual podemos importar essas configurações e utilizá-las em nossos códigos.

Comentários

Linha 2: Atributo que indica, com valores booleanos, se a aplicação está sendo executada em ambiente de produção ou não. Este atributo não pode ser apagado, já que o mesmo é utilizado pelo Angular. 

production: false,

Linha 3: Ao atributo lembretesApiUrl atribuímos o endereço da API de lembretes utilizada pela aplicação. 

lembretesApiUrl: 'https://devreminder.herokuapp.com'
