Visão geral

Na classe do componente form-lembrete, chamada de FormLembreteComponent, colocamos a entrada e saída de dados do componente. Esses dados tem como origem e destino o componente pai (ou o componente da página) que estiver instanciando o nosso componente de formulário.

Comentários

Linhas 1 e 2: Realizamos a importação dos decorators Component, Input e Output, bem como do EventEmitter, que será utilizado para enviar dados do componente de formulário para o componente pai, e da interface Lembrete. 

01 import [curly_open] Component, Input, Output, 
   EventEmitter [curly_close] from '@angular/core';
02 import [curly_open] Lembrete [curly_close] 
   from '../../interfaces/lembrete';

Linhas 4 a 8: Logo abaixo das importações encontraremos o Decorator. O Angular CLI terá criado o decorator @Component automaticamente, junto com seus três atributos: selector, templateUrl e styleUrls. 

@Component([curly_open]
  selector: 'app-form-lembrete',
  templateUrl: './form-lembrete.component.html',
  styleUrls: ['./form-lembrete.component.css']
[curly_close])
  • selector: é o identificador do componente. Com ele podemos utilizar o componente dentro de outros componentes, como uma tag.
  • templateUrl: indica o caminho para o arquivo HTML onde ficará a formatação do nosso componente.
  • styleUrls: indica o caminho para o arquivo CSS de estilo do nosso componente. Os estilos definidos nesse arquivo só estarão disponíveis no arquivo .html definido no atributo templateUrl.

Linha 10: Criamos um atributo com o decorator @Input, tornando possível receber um lembrete do componente pai (ou componente da página). 

@Input() lembrete: Lembrete = [curly_open][curly_close];

Linha 11: Criamos um atributo com o decorator Output e criamos uma instância do EventEmitter. Dessa forma podemos, sempre que precisarmos, emitir um evento que executará um método do componente pai (ou componente da página) que for passado no atributo outputLembrete. 

@Output() outputLembrete: EventEmitter = new EventEmitter();

Com os atributos decorados com @Input e Output seremos capazes de enviar e receber dados do componente de formulário a partir dos componentes das páginas de criação e edição de lembretes. Podemos ver claramente a utilização desses atributos nos templates dos componentes citados anteriormente: componente da página de criação e componente da página de edição

Linha 13, 14 e 15: O método onSubmit, único método do nosso componente, será executado sempre que o formulário do componente for submetido. Dessa forma podemos utilizar o método emit do EventEmitter para executar um método do componente pai. 

onSubmit() [curly_open]
 this.outputLembrete.emit(this.lembrete);
[curly_close]
