Visão geral

Nesse módulo, criado automaticamente pelo Angular CLI, definiremos as rotas da nossa aplicação. Isto é, definiremos que para uma dada URI acessada pelo usuário, um determinado componente (de página) deve ser carregado.

Comentários

Linha 1: O módulo NgModule permitirá o uso de um decorator que indica pro Angular que a classe AppRoutingModule se trata de um módulo. 

import [curly_open] NgModule [curly_close] from '@angular/core';

Linha 2: Importamos o módulo RouterModule e a interface Routes que serão utilizados para realizar, de fato, o roteamento da aplicação e a tipagem do objeto de configuração da rota. 

import [curly_open] Routes, RouterModule [curly_close] from '@angular/router';

Linhas 3, 4 e 5: Importamos os componentes das páginas da nossa aplicação. 

03  import [curly_open] ListaLembreteComponent [curly_close] 
    from './paginas/lista-lembrete/lista-lembrete.component';
04  import [curly_open] CriarLembreteComponent [curly_close] 
    from './paginas/criar-lembrete/criar-lembrete.component';
05  import [curly_open] EditarLembreteComponent [curly_close] 
    from './paginas/editar-lembrete/editar-lembrete.component';

Linha 7 a 12: Definimos cada uma das rotas de nossa aplicação e seus respectivos componentes. Para a rota de edição teremos teremos, também, a presença de um parâmetro chamado id. Além disso, para qualquer outra rota desconhecida, redirecionaremos o usuário para a página inicial. 

const routes: Routes = [
  [curly_open]path: '', component: ListaLembreteComponent[curly_close],
  [curly_open]path: 'lembrete/criar', component: EditarLembreteComponent[curly_close],
  [curly_open]path: 'lembrete/editar/:id', component: CriarLembreteComponent[curly_close],
  [curly_open]path: '**', redirectTo: ''[curly_close]
];

Linhas 14 a 17: No decorator NgModule iniciamos o roteamento da aplicação com o método forRoot e exportamos o módulo criado. 

@NgModule([curly_open]
  imports: [RouterModule.forRoot(routes)],
  exports: [RouterModule]
[curly_close])
