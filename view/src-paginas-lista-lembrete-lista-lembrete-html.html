Visão geral

Nesse template criaremos a formatação da listagem de lembretes que será exibida para o usuário. Nessa página encontraremos, também, botões para criar um novo lembrete (que redirecionará para a página de criação) e um botões para deletar lembretes.

Comentários

Linhas 3 e 4: Em uma coluna de tamanho 8 e centralizada criamos uma instância do componente de mensagem de erros. Dessa forma, quando o método setError do componente de mensagem de erros for executado na classe do componente do de listagem de lembretes, a mensagem de erro será exibida nessa posição. 

<div class="col-md-8 mx-auto">
 <app-error-msg></app-error-msg>

Linha 5, 6 e 7: Ainda na coluna centralizada, criamos um botão à direita com o atributo routerLink que, ao clicar no botão, redirecionará o usuário para a página de criação de lembretes. 

01  <div class="text-right">
02   <button type="button" class="btn btn-success" routerLink="
     /lembrete/criar">Novo lembrete</button>
03  </div>

Linhas 8, 9 e 10: Utilizamos o método existemLembretes, da classe do componente, para verificar se existem lembretes. Caso não exista nenhum, uma mensagem será exibida para o usuário. 

<div class="text-center" *ngIf="!existemLembretes()">
 Nenhum lembrete cadastrado.
</div>

Linha 13: Utilizamos o atributo *ngFor para criar uma linha (row, do Bootstrap) para cada lembrete da lista armazenada no atributo lembretes. Dentro desse elemento encontraremos o card Bootstrap referente ao lembrete dessa interação do laço. 

<div class="row mt-3" *ngFor="let lembrete of lembretes">

Linhas 16 a 23: No cabeçalho do card criaremos um badge com o nível de prioridade do usuário. Além disso, a cor desse badge será definida de acordo com o nível de prioridade. 

<div class="card-header">
  <span class="badge badge-pill" 
   [class.badge-danger]="lembrete.prioridade == 'ALTA' " 
   [class.badge-warning]="lembrete.prioridade == 'MEDIA' "
   [class.badge-success]="lembrete.prioridade == 'BAIXA' ">
   PRIORIDADE [curly_open][curly_open]lembrete.prioridade[curly_close][curly_close]
 </span>
</div>

Linhas 24, 25 e 26: No corpo do card realizaremos a impressão do conteúdo do bilhete. 

<div class="card-body">
 <p class="card-text">[curly_open][curly_open]lembrete.conteudo[curly_close][curly_close]</p>
</div>

Linhas 28 a 31: Finalmente, no rodapé do card, criaremos dois botões: o primeiro redirecionará o usuário para a rota de edição de lembretes, passando o id numérico desse lembrete como parâmetro. O segundo, ao ser clicado, executará o método da deletaLembrete, que ficará responsável por excluir o lembrete e atualizar a listagem de lembretes. 

28  <button type="button" class="btn btn-primary btn-sm ml-1" routerLink="/lembrete
    /editar/[curly_open][curly_open]lembrete.id[curly_close][curly_close]">Editar</button>
31  <button type="button" class="btn btn-danger btn-sm ml-1" 
    (click)="deletaLembrete(lembrete.id)">X</button>
