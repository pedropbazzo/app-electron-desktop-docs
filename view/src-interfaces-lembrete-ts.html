src/interfaces/lembrete.ts 

type prioridade = 'BAIXA' | 'MEDIA' | 'ALTA';
export interface Lembrete [curly_open]
    id: number;
    conteudo: string;
    arquivado: boolean;
    prioridade: prioridade;
    modificado: number;
[curly_close]

Visão geral

Nesta interface definimos a estrutura da entidade lembrete, seguindo a mesma estrutura de resposta da API de lembretes. Essa interface será utilizada por toda a aplicação auxiliando o desenvolvimento da mesma. Entretanto, ao construirmos a aplicação, esta interface deixará de existir.

Comentários

Linha 1: Criamos um novo tipo chamado prioridade. Este tipo só pode receber um dos três seguintes valores: BAIXA, MÉDIA e ALTA. 

type prioridade = 'BAIXA' | 'MEDIA' | 'ALTA';

Linha 2: Criamos e exportamos a interface Lembrete. Exportá-la é necessário para que esta possa ser utilizada em outros pontos da aplicação. 

export interface Lembrete [curly_open]

Linha 3: O atributo id receberá o identificador único e numérico do lembrete. Este valor é gerado pela API durante a inserção de um lembrete. 

id: number;

Linha 4: No atributo conteudo será atribuído o lembrete em si, uma fase que nos permita lembrar da atividade a ser realizada. 

conteudo: string;

Linha 5: Este atributo, embora não seja utilizado pela nossa aplicação, está presente na resposta da API. Ele poderia ser utilizado para criar funcionalidades do soft delete, por exemplo. 

arquivado: boolean;

Linha 6: No atributo prioridade definimos que seus valores devem ser do tipo prioridade criado na linha 1. Isto é, deve ser a string BAIXA, MÉDIA ou ALTA. 

prioridade: prioridade;

Linha 7: O atributo modificado é armazenada a data desde a ultima atualização (ou inserção) do lembrete. Esta data é criada automaticamente pela API no momento de inserção ou atualização do lembrete e está no formato UNIX, isto é, um número inteiro que representa a contagem do tempo recorrido em segundos desde 01 de Janeiro de 1970. 

modificado: number;
